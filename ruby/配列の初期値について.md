# 配列の初期値について

`Array.new` を使用する際、

初期値を第2引数に指定する場合とブロックに指定する場合で挙動が違います。

<br>

## 初期値を第2引数で指定する場合

`Array.new(3, "a")` とした場合、 `"a"` オブジェクトは1つしか生成されず、

配列の全要素が、同じ `"a"` オブジェクトを参照します。

### 確認

```
array = Array.new(3, "a")
array[0].succ!

p array
```

⇒ `["b", "b", "b"]`

配列の0番目のみを変更したつもりが、全ての要素が変更されました。

<br>

## 初期値をブロックで指定する場合

`Array.new(3) { "a" }` とした場合、 `"a"` オブジェクトは要素の数だけ生成され、

配列の各要素で、異なる `"a"` オブジェクトを参照します。

### 確認

```
array = Array.new(3) { "a" }
array[0].succ!

p array
```

⇒ `["b", "a", "a"]`

配列の0番目のみが変更されました。
